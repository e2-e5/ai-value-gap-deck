# Structure and narrative direction

## Suggested short deck structure

1. **Cover** — AI value gap is not a model problem
2. **Signal** — adoption vs value realization gap
3. **Diagnosis** — the operating model is broken
4. **Why wrappers fail** — local optimization, no moat, shallow integration
5. **Why industrializers win** — deeper ownership, standards, governance, compounding capability
6. **AI as infrastructure** — stack view
7. **CEO/founder mandate** — what leadership should actually do
8. **90-day agenda** — practical next step
9. **Close** — wrappers capture attention; infrastructure captures value

## Visual direction for current v0

Chosen direction: **dark Swiss / infra-modern hybrid**

Why:
- serious enough for CEO audience
- still feels contemporary / AI-native
- avoids over-designed “AI neon demo” aesthetics
- gives designer room to either go more premium editorial or more product/system-like later

## Design principles

- one claim per slide
- low-density typography, high contrast
- infrastructure metaphors over chatbot metaphors
- visible component states in cards/rails for static screenshot quality
- reusable blocks, not one-off slide styling

## What the designer can safely change later

- typography pairing
- accent palette
- hero composition
- layout ratios
- component styling inside `styles/blocks.css`

## What should probably remain stable

- core story arc
- 8–9 slide length
- infrastructure framing
- anti-wrapper thesis
- CEO call to action
